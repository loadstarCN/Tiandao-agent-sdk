# 天道开发指南

> 面向开发者的日常操作手册：启动环境、验证 agent、跑测试、闭环改代码。

---

## 一、开发环境

### 启动所有服务

```powershell
cd D:\Workshop\Tiandao
.\dev.ps1          # 启动全部服务（纯文字日志，兼容任何终端）
.\dev.ps1 tui      # 启动并显示 TUI 面板（需要 Windows Terminal）
```

五个服务按依赖顺序自动启动：

```
world-engine (:8080) → ai-service (:8081) → observer-ui (:3001)
                                           → agent-qingsong
                                           → agent-yunlanzi
```

### 停止 / 重启

```powershell
.\dev.ps1 down                     # 停止全部
.\dev.ps1 process restart ai-service   # 重启单个服务
```

### 查看日志

```powershell
.\dev.ps1 logs                     # 聚合日志流
.\dev.ps1 logs agent-qingsong      # 单服务日志
```

### 健康检查

```
GET http://localhost:8080/health   → "天道运转，生生不息"
GET http://localhost:8081/health   → {"status":"ok","service":"narrative"}
GET http://localhost:3001          → 200
```

### 首次安装依赖

```powershell
# process-compose（进程管理）+ cargo-watch（Rust热重载）
.\scripts\install-dev-tools.ps1
```

## 二、验证 Agent 状态

### 方式 1：实时监控面板（推荐）

```powershell
# 每2秒刷新，显示所有修仙者 + 最近叙事
python scripts/watch_agents.py

# 查特定修仙者详情
python scripts/watch_agents.py --name 青松道人

# 查最近叙事历史（默认20条）
python scripts/watch_agents.py --history --limit 50
```

### 方式 2：Observer UI

浏览器打开 `http://localhost:3001`，可视化全局地图和叙事流。

### 方式 3：直接调 API

> **注意**：observe 端点需要 `X-Internal-Key` 认证头（值为 INTERNAL_API_KEY 环境变量）。

```bash
# 所有修仙者列表
curl -H "X-Internal-Key: $INTERNAL_API_KEY" http://localhost:8080/v1/observe/agents

# 最近叙事事件
curl -H "X-Internal-Key: $INTERNAL_API_KEY" "http://localhost:8080/v1/observe/history?limit=20"

# 特定修仙者状态
curl -H "X-Internal-Key: $INTERNAL_API_KEY" "http://localhost:8080/v1/observe/status?agent_id=<UUID>"

# 实时 SSE 叙事流
curl -N -H "X-Internal-Key: $INTERNAL_API_KEY" http://localhost:8080/v1/observe/stream
```

### 方式 4：process-compose 日志

```powershell
# 看 agent 的决策过程（感知→思考→行动）
.\dev.ps1 logs agent-qingsong

# 看叙事服务生成情况
.\dev.ps1 logs ai-service
```

### 发传音（人工干预测试）

```bash
# 先获取 agent_id，再发传音
curl -X POST http://localhost:8080/v1/observe/whisper \
  -H "Content-Type: application/json" \
  -H "X-Internal-Key: $INTERNAL_API_KEY" \
  -d '{"agent_id":"<UUID>","content":"去北边看看"}'
```

## 四、接入真实 Agent（OpenClaw 等）

天道通过 TAP 协议接入，推荐使用 tiandao-cli：

### CLI + MCP Server

```bash
pip install tiandao-cli

# CLI 模式
tiandao login --token "your-tap-token"
tiandao perceive
tiandao act --action-type cultivate --intent "感悟天地灵气"

# MCP Server 模式（stdio）
python -m tiandao_cli
```

在 Claude Code / Claude Desktop / OpenClaw 的配置文件里添加：

```json
{
  "mcpServers": {
    "tiandao": {
      "command": "python",
      "args": ["-m", "tiandao_cli"],
      "env": {
        "TAP_TOKEN": "<注册后获得的JWT>"
      }
    }
  }
}
```

MCP 工具：`tiandao_perceive` / `tiandao_act` / `tiandao_world_guide` / `tiandao_whisper`（注册修仙者请通过 [tiandao.co](https://tiandao.co) 门户完成）

## 五、代码改动闭环

修改代码后无需手动重启——`cargo-watch` 和 `uvicorn --reload` 自动处理热重载：

| 改动 | 自动触发 | 延迟 |
|---|---|---|
| `.rs` 文件变动 | cargo 重编译 + 重启 world-engine | ~3-5s |
| `.py` 文件变动 | uvicorn 重载 ai-service | ~1s |
| `.tsx` 文件变动 | Next.js HMR，浏览器自动刷新 | ~1s |

**推荐流程：**

```
发现问题
  ↓
写测试（tests/ 或 world-engine/tests/）
  ↓
python -m pytest <具体测试> -v     ← 确认失败
  ↓
改代码
  ↓
cargo-watch / uvicorn 自动重载
  ↓
python -m pytest <具体测试> -v     ← 确认通过
```
